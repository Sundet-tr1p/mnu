# UniConnect

**Полное руководство по коду, архитектуре, API, i18n и всем функциям:** [docs/UNICONNECT_PROJECT_GUIDE.md](docs/UNICONNECT_PROJECT_GUIDE.md)

## Описание проекта

**UniConnect** — это внутренняя университетская платформа для MNU, объединяющая функции студенческого портала и социальной сети. Платформа доступна только для пользователей с email `@kazguu.kz` или `@mnu.kz`.

## Возможности

- Лента новостей с постами, лайками и комментариями
- Организации (студенческие клубы)
- Чаты (глобальный, по школам и специальностям)
- Отзывы на преподавателей (только для студентов)
- FAQ (аккордеон)
- Опросы (ссылки на внешние формы)
- Уведомления (лайки, комментарии, сообщения)
- Профиль пользователя с возможностью редактирования

## Технический стек

- **Frontend:** Next.js 14 (App Router), React, TypeScript, Tailwind CSS
- **Backend:** Next.js API Routes
- **Database:** MySQL
- **ORM:** Prisma
- **Аутентификация:** JWT + HttpOnly cookies
- **Валидация:** Zod

## Установка

1. Клонируйте репозиторий:
   ```
   git clone <URL>
   ```
2. Перейдите в директорию проекта:
   ```
   cd uniconnect
   ```
3. Установите зависимости:
   ```
   npm install
   ```
4. Настройте переменные окружения, скопировав `.env.example` в `.env.local` и заполнив необходимые значения.
5. Запустите миграции базы данных:
   ```
   npx prisma migrate dev --name init
   ```
6. Заполните базу данных тестовыми данными:
   ```
   npm run db:seed
   ```
7. Запустите проект в режиме разработки:
   ```
   npm run dev
   ```

## Примечание для администратора
- **Где менять язык/тексты:** `src/lib/i18n.ts` (все надписи KZ/RU/EN).
- **Где менять меню и пункты навигации:** `src/components/Sidebar.tsx` и `src/components/layout/MobileNav.tsx`.
- **Где менять правила входа/домен email:** `.env.local` (`ALLOWED_EMAIL_DOMAIN`) и `src/lib/validators.ts`.
- **Где менять визуальный стиль:** `src/app/globals.css` (градиенты, анимации, glass-эффекты) и `src/components/ui/*`.
- **Где менять тестовых пользователей и стартовые данные:** `prisma/seed.ts`, после изменений выполнить `npm run db:seed`.
- **Где менять подключение к БД и секреты:** только в `.env.local` (`DATABASE_URL`, `JWT_SECRET`), не хранить секреты в репозитории.

### Быстрый чек при ошибке в dev-режиме
Если появляется ошибка вида `Cannot find module './xx.js'`, это обычно кэш Next.js:
1. Остановить сервер.
2. Удалить папку `.next`.
3. Запустить снова: `npm run dev`.
